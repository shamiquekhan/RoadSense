Strengths
Dual scoring engine — 4-component + 3-module RoadSense pipeline gives evaluators two independent methodologies to review
CLIP fallback for Mapillary imagery (open-clip-torch) — genuinely innovative for countries with sparse street-level coverage
OSM enrichment via both Overpass API and pyrosm PBF — great for replicability across Asia-Pacific countries with no proprietary data
Config-driven scoring weights (scoring_weights.yaml) makes assumptions explicit and reproducible — exactly what judges want to see
4-tier output (Critical / High / Medium / Low) with plain-language labels designed for transport ministry audiences
Tests folder + CI via GitHub Actions + Makefile — signals production-grade engineering discipline
Critical gaps before 25 Jun deadline
No live map URL. The challenge explicitly asks for a working URL if submitting an interactive map. Hosting on GitHub Pages or Vercel is required — folium output alone is not enough.
Data citations are a placeholder. The README literally says "Add the exact citations for each project dataset here." Judges checking the repo will see this immediately.
README setup instructions are broken. Run commands reference src.preprocessing but the package is in src/roadsense/. The referenced environment.yml doesn't appear to exist. A judge who can't run the code can't grade it.
No committed outputs or sample data. The evaluation criteria requires documented results. Even a small synthetic demo dataset + pre-generated outputs would show the pipeline actually runs.
Only 2 commits. The bulk-upload history makes the project look newly created. A commit history with logical stages (preprocessing → modules → scoring → visualisation) builds reviewer confidence.
4-component scoring undocumented. The pipeline runs two approaches but only the 3-module RoadSense approach is explained in the README. Judges need to understand both.
Minor: No GitHub repo description or topics. No architecture diagram rendered in README (SVG may not load for all viewers). No screenshot of the output map.
